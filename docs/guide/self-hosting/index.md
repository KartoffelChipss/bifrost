# Self Hosting

Bifröst is designed to be self-hosted, giving you full control over your deployment.

You can run Bifröst on your own server using Docker or manually, depending on your preference and technical expertise. This allows you to customize the bridge to your needs and ensures that your data remains private.

## Self-Hosting Options

You can self-host Bifröst using Docker on Unix-based systems (Linux and macOS) or manually on Windows. Choose the option that best fits your environment and technical expertise:

### Linux and macOS

- ⭐ [Self-Hosting Bifröst using Docker on Unix-based Systems](./linuxmacos/docker.md)
- 🔧 [Self-Hosting Bifröst Manually on Unix-based Systems](./linuxmacos/manual.md)

### Windows

- ⭐ [Self-Hosting Bifröst using Docker on Windows](./windows/docker.md)
- 🔧 [Self-Hosting Bifröst Manually on Windows](./windows/manual.md)

Guides marked with a ⭐ are recommended for most users, as they provide an easier setup and better performance if you have Docker available.

Guides marked with a 🔧 are for users who prefer or need to run Bifröst without Docker, but they require more technical knowledge and may involve a more complex setup process.
